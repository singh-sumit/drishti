# QEMU Validation Lane

This directory contains Drishti v0.4 cross-arch QEMU smoke tooling.

## Prerequisites

Required commands:

- `qemu-system-x86_64`
- `qemu-system-aarch64` (for arm64 lane)
- `cpio`, `gzip`, `file`
- `rustup` with targets:
  - `x86_64-unknown-linux-gnu` (default host target on amd64)
  - `aarch64-unknown-linux-musl` for arm64 lane

Build artifacts required before `prepare.sh`:

- `drishti-daemon` with `ebpf-runtime` feature
- `qemu_smoke` guest harness binary

Use `xtask` to build and prepare deterministically:

```bash
cargo run -p xtask -- qemu prepare --arch x86_64
cargo run -p xtask -- qemu smoke --arch x86_64
```

If `/boot/vmlinuz-*` is unavailable (common on WSL), `prepare.sh` falls back to
downloading and extracting the current Ubuntu generic x86_64 kernel package.

## aarch64 Kernel Input

For aarch64 runs, provide a kernel explicitly:

- `DRISHTI_QEMU_AARCH64_KERNEL=/abs/path/to/vmlinuz`
- or `DRISHTI_QEMU_AARCH64_KERNEL_DEB=/abs/path/to/linux-image-arm64.deb`

If the arm64 Rust target is missing, `xtask` fails fast with:

```bash
rustup target add aarch64-unknown-linux-musl
```

## Outputs

All run artifacts are written under `target/qemu/<arch>/`:

- `serial.log`
- `smoke.log`
- `metrics.prom`
- `summary.json`
- `index.json` (generated by `xtask qemu ci`)

Note: serial logs can include `Kernel panic - not syncing: Attempted to kill init!`
after a successful smoke result. This is expected in the minimal initramfs model when
`/init` exits and does not indicate a failed run by itself.
